Why MN SkillVault exists

A front end for a folder that already had everything

MN SkillVault doesn't store any lesson content itself. It reads a structure that already exists in Google Drive and gives it a shape people can actually browse.

Mission

Make the tech-skill playlists already sitting in a Drive folder as easy to explore as a real course catalog — searchable, categorized, and honest about where the content actually lives.

Vision

A learning hub that never fights its own content: adding a new skill is as simple as dropping a folder into Drive, and the catalog can reflect it without a redeploy.

Under the hood

Why a Drive-backed front end is efficient

There's no database to maintain and no file storage bill — Drive already does that job.

Zero hosting cost for content

Every video and document stays in Drive. The site only holds links and metadata, so it never needs its own storage budget.

Always up to date

Add a file to the Drive folder and it's live the moment the catalog entry is added — no publishing pipeline in between.

One source of truth

data/courses.json mirrors the Drive structure, so the catalog and the folder never quietly drift apart.

Familiar permissions model

Access control rides on Drive sharing settings that already exist, instead of a second login system to keep secure.

In practice

Publishing a new playlist

1. Create a folder in Drive

Group the videos, slides, and notes for a skill into one shared folder.

2. Add one entry to courses.json

Title, category, description, tags, and the folder's share link — that's the entire integration.

3. Refresh the Courses page

courses.js reads the file on load, so the new playlist appears with search and filtering already working.

Behind the scenes

Admin panel

MN SkillVault is a static front end, so "admin access" means controlling two things: who can edit the Drive folder, and who can edit courses.json. There's no live login system in this version — the mock below shows the shape a future one would take.

What an admin actually controls

  • Add, rename, or remove playlist folders in Drive
  • Edit data/courses.json to publish or update a catalog entry
  • Manage Drive sharing permissions for viewers vs. editors
  • Review messages submitted on the Contact page
This screen is a visual mock only. Wiring up real authentication needs a backend (for example Firebase Auth or a small API) — plug it in behind this same layout when you're ready to add one.
